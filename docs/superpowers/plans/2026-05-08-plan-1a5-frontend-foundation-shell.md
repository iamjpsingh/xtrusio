# Plan 1A.5 — Frontend Foundation & Layout Shell Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the visual foundation of the Xtrusio web app — Tailwind v4, shadcn-ui, TanStack Router, B/W theme tokens (light + dark + status colors), Geist typography, layout shell (sidebar + topbar), and 5 routes with real empty states. No mock data, no custom CSS outside `globals.css`, TypeScript only.

**Architecture:** Single-page Vite + React 19 + TypeScript app. Routing is file-based via TanStack Router. Styling is Tailwind v4 (CSS-first config) on top of shadcn-ui components owned in-tree. Theme switching via `next-themes`. All components are stateless prop-driven, theme-agnostic, ≤ 250 LoC.

**Tech Stack:** Tailwind v4, `@tanstack/react-router`, shadcn-ui (CLI), `next-themes`, Geist (via `@fontsource-variable/geist`), `lucide-react`, `sonner`. All TypeScript; configs in `.ts` (no `.js`/`.jsx`/`.mjs`).

  const [open, setOpen] = useState(false);

  useEffect(() => {
    const onKey = (e: KeyboardEvent) => {

## Task 26: Color-lint script + pre-commit hook

**Files:**
- Create: `scripts/check-colors.sh`
- Modify: `.pre-commit-config.yaml`

- [ ] **Step 1: Create `scripts/check-colors.sh`**

```bash
#!/usr/bin/env bash
# Block hardcoded colors and Tailwind palette utilities in frontend paths.
# Enforces ENGINEERING_PRINCIPLES section 2.0 + spec #4 section 9.
set -euo pipefail

# Files to scan: only newly added/modified frontend source under apps/web or packages/ui
staged=$(git diff --cached --name-only --diff-filter=AM \
  | grep -E '^(apps/web|packages/(ui|api-types))/.*\.(ts|tsx|css)$' \
  | grep -v 'src/globals\.css$' \
  | grep -v '/components/ui/' \
  | grep -v '/routeTree\.gen\.ts$' \
  || true)

if [ -z "$staged" ]; then
  exit 0
fi

# Patterns to forbid:
#   - Hex colors (#abc, #abcdef, #abcdef00)
#   - Tailwind palette utilities for colors that should go through tokens
forbidden_patterns=(
  '#[0-9a-fA-F]{3,8}\b'
  '(bg|text|border|ring|fill|stroke|from|via|to)-(slate|zinc|neutral|stone|red|orange|amber|yellow|lime|green|emerald|teal|cyan|sky|blue|indigo|violet|purple|fuchsia|pink|rose)-[0-9]+'
)

violations=0
for file in $staged; do
  for pat in "${forbidden_patterns[@]}"; do
    matches=$(grep -nE "$pat" "$file" 2>/dev/null || true)
    if [ -n "$matches" ]; then
      echo "ERROR: hardcoded color in $file (use semantic tokens, see ENGINEERING_PRINCIPLES section 2.0):"
      echo "$matches" | sed 's/^/  /'
      violations=$((violations + 1))
    fi
  done
done

if [ "$violations" -gt 0 ]; then
  exit 1
fi
exit 0
```

## Notes for the Engineer

- **Why TDD on EmptyState specifically?** Because it's the one component that holds genuine logic (disabled-with-tooltip vs not, action click handler). Routes and other shadcn-composed components are mostly chrome — they get a smoke test (Task 25) but not full TDD.
- **Why `routeTree.gen.ts` is committed:** PR diffs need to show route changes. Treat it like a generated lock file — it's reproducible but you want it under review.
- **Why `tailwindcss-animate` isn't here:** Tailwind v4 has `animate-*` built in. shadcn's recent versions support v4 without the plugin. If the `shadcn add sidebar` step in Task 7 errors about animations, install the plugin (`pnpm add tailwindcss-animate`) and add `@plugin "tailwindcss-animate";` to `globals.css`.
- **Why `disableTransitionOnChange` on `ThemeProvider`:** prevents flash of cross-fade animations when the OS-level theme changes — Linear does this, looks more polished.
- **Don't skip Task 28.** A working `make check` doesn't mean the visual app actually works. Eyes on the browser is mandatory before claiming done.
